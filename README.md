# Stop Doom Scrolling — Chrome Extension

A Chrome extension that fights back against mindless doom scrolling on **Facebook** and **Instagram**
by injecting a fake post from the account **@stop_doomscrolling_bitch** right into your feed.

---

## What it does

The extension watches how you scroll and fires a reality-check post when **any** of these happen:

| Trigger | When |
|---------|------|
| **Time limit** | You've been on the site for **7 minutes** |
| **Page bottom** | You reach the bottom of the feed (autoload zone) |
| **Post count** | Every **5–10 new posts** that load while you scroll |

The fake post looks like a real feed card and contains:
- A bold canvas-drawn **"STOP DOOM SCROLLING"** image
- Account name **stop_doomscrolling_bitch**
- A randomly chosen **suggestion** of something better to do
- A dismiss button so you can keep scrolling if you really want to (you weak, beautiful person)

## Installation

### Step 1 — Generate the icons (one-time setup)

The extension needs PNG icon files. Run the bundled Python script to create them:

```bash
cd doom-scroll-stopper
python generate_icons.py
```

This creates `icons/icon16.png`, `icons/icon48.png`, and `icons/icon128.png`.
No external packages required — pure Python standard library.

> **Alternative:** You can put any three PNG files named `icon16.png`, `icon48.png`,
> `icon128.png` into the `icons/` folder if you'd prefer your own artwork.

---

### Step 2 — Load the extension in Chrome

1. Open Chrome and go to `chrome://extensions`
2. Enable **Developer mode** (toggle in the top-right corner)
3. Click **"Load unpacked"**
4. Select the `doom-scroll-stopper/` folder

The extension icon (a red stop-sign circle) should appear in your toolbar.

> **Note:** Every time you edit any file in the extension folder, click the
> **refresh icon** on the extension card at `chrome://extensions` to reload it.

## Customising suggestions

There are **two ways** to edit the suggestions list:

### Option A — Edit `suggestions.json` (permanent defaults)

Open `suggestions.json` in any text editor and add, remove, or change lines.
Each entry is a JSON string in an array:

```json
[
  "Go for a 10-minute walk 🚶",
  "Your custom suggestion here",
  "Another one..."
]
```

After saving, **reload the extension** at `chrome://extensions`.

### Option B — Use the popup editor (saved in browser storage)

Click the extension icon in the Chrome toolbar to open the popup.

- The **textarea** shows all current suggestions (one per line).
- Edit directly, then click **Save**.
- Click **Reset defaults** to wipe your changes and go back to `suggestions.json` defaults.

> Popup changes take priority over `suggestions.json` and sync across Chrome profiles
> via `chrome.storage.sync`.

## Popup controls

| Control | What it does |
|---------|-------------|
| **Enable/Disable toggle** | Turns the extension on or off without uninstalling |
| **Reality checks delivered** | Counter of how many posts have been shown this session |
| **Suggestions textarea** | Edit the suggestion pool directly |
| **Save** | Stores your changes in Chrome sync storage |
| **Reset defaults** | Reverts to the built-in suggestions list |
| **Preview post** | Forces a doom post into the current Facebook/Instagram tab |

## Troubleshooting

**The post doesn't appear**
- Make sure you're on the main feed page (`facebook.com/` or `instagram.com/`), not a single post or story.
- Check that the extension is enabled in the popup.
- Instagram and Facebook update their DOM structure periodically — if selectors break, open an issue.

**Preview button says "Could not reach page"**
- Refresh the Facebook/Instagram tab and try again.
- The content script only runs after the page fully loads.

**Icons are missing / extension won't load**
- Run `python generate_icons.py` from inside the `doom-scroll-stopper/` folder.
- Make sure the `icons/` directory exists with the three `.png` files before loading the extension.

---

## Notes

- The extension uses **Shadow DOM** for its fake post so Instagram/Facebook CSS can't
  interfere with it (and vice versa).
- Images are generated with the browser's **Canvas API** at runtime, avoiding
  Content Security Policy issues with external images.
- No data leaves your browser. Zero analytics, zero tracking.
